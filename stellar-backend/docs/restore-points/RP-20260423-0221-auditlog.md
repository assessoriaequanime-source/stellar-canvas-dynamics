# Fase AuditLog – Trilha de Auditoria

## Objetivo
Implementar, validar e documentar o módulo de trilha de auditoria (AuditLog) para registrar ações sensíveis do usuário, garantindo rastreabilidade e segurança.

## O que foi feito
- Schema Prisma para AuditLog e enum AuditAction
- Serviço createAuditLog
- Rotas protegidas /api/v1/audit
- Integração de logs em consentimento e transações
- Testes automatizados cobrindo logs e isolamento por usuário
- Checklist de build, lint e testes

## Arquivos criados
- prisma/schema.prisma (enum AuditAction, tabela auditLog)
- src/lib/audit.ts
- src/api/routes/audit.ts
- src/api/validators/audit.ts

## Arquivos alterados
- src/api/routes/consent.ts (integração AuditLog)
- src/api/routes/transaction.ts (integração AuditLog)
- tests/integration/api.integration.test.ts (testes de auditoria)
